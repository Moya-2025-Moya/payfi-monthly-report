# StablePulse — 稳定币行业原子化知识引擎

## 一句话定义

把全网稳定币信息拆成**原子事实**，按任意维度自由重组，自动检测事实矛盾和知识盲区——团队在这个自动生长的事实底座上协作标注、讨论、跨周追踪。

---

## 核心设计原则

| 原则 | 含义 |
|---|---|
| **AI零主观** | AI只做：拆事实、识别实体、串时间线、检测矛盾、对比结构。永远不说"该投/不该投"，永远不输出观点 |
| **原子事实是最小单位** | 系统不存储"文章"，而是把每条信息拆成可追溯、可查询、可对比的原子事实。这是架构级的代差 |
| **多维度独立验证** | 每条事实经6个独立验证员检验（来源回溯、多源交叉、数值合理性、链上锚定、时序一致、时间线归属），投票制裁决，少数服从多数。不依赖人工验证 |
| **全面不遗漏** | 不帮你筛信息，而是把所有碎片归到正确的位置。同一类信息多数据源采集，为交叉验证提供基础 |
| **系统知道自己不知道什么** | 主动暴露信息缺口和事实冲突，比假装什么都知道更有价值 |
| **团队集体记忆** | 笔记、标记、讨论沉淀在原子事实上，成为跨周持续生长的集体智慧 |

## 核心架构：原子事实引擎

### 什么是原子事实

一篇新闻 = 多个原子事实。每个原子事实是一个**不可再拆、可独立验证、可追溯来源**的信息单元。

```
原始新闻: "Circle提交了S-1修订版，披露年收入$1.7B"
                          │
                     AI拆解为:
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
  ┌─────▼─────┐    ┌─────▼─────┐    ┌─────▼─────┐
  │ 原子事实#1 │    │ 原子事实#2 │    │ 原子事实#3 │
  │            │    │            │    │            │
  │ Circle于   │    │ Circle年   │    │ 其中78%   │
  │ 3/8提交    │    │ 收入$1.7B  │    │ 来自储备  │
  │ S-1修订版  │    │            │    │ 利息收入  │
  │            │    │            │    │            │
  │ 实体:Circle│    │ 实体:Circle│    │ 实体:Circle│
  │ 类型:监管  │    │ 类型:财务  │    │ 类型:财务  │
  │ 标签:[IPO, │    │ 标签:[收入,│    │ 标签:[收入,│
  │  SEC,S-1]  │    │  财报]     │    │  利息,储备]│
  │ 来源:SEC   │    │ 来源:SEC   │    │ 来源:SEC   │
  │ 可信度:官方│    │ 可信度:官方│    │ 可信度:官方│
  └────────────┘    └────────────┘    └────────────┘
```

### 原子事实的Schema

```typescript
interface AtomicFact {
  id: string
  // 内容
  content_en: string              // "Circle reported annual revenue of $1.7B"
  content_zh: string              // "Circle披露年收入$1.7B"
  // 溯源
  source_id: string               // 指向raw_*表的原始数据
  source_type: 'news' | 'filing' | 'onchain' | 'product' | 'funding' | 'tweet' | 'regulatory'
  source_url: string              // 原文链接
  source_credibility: 'official' | 'media' | 'social' | 'derived'
  // 分类
  fact_type: 'event' | 'metric' | 'quote' | 'relationship' | 'status_change'
  tags: string[]                  // ['revenue', 'IPO', 'S-1']
  // 关联
  entity_ids: string[]            // 关联的实体
  sector_ids: string[]            // 关联的赛道
  timeline_id?: string            // 所属时间线
  // 数据 (如果是metric类型)
  metric_name?: string            // 'annual_revenue'
  metric_value?: number           // 1700000000
  metric_unit?: string            // 'USD'
  metric_period?: string          // '2025-FY'
  metric_change?: string          // '+23% YoY'
  // 时间
  fact_date: Date                 // 事实发生时间
  collected_at: Date              // 采集时间
  week_number: string             // '2026-W10'
}
```

### 多维度验证架构

每条原子事实提取后，必须经过完整验证流水线才能进入系统：

```
原始数据 → B1提取(两次prompt交叉自查)
         → V1-V5 五个验证员并行验证
         → V0 纯代码裁决器汇总投票
         → 只有通过验证的事实进入知识引擎
```

**6个独立验证员**:

| 验证员 | 职责 | 实现 |
|---|---|---|
| V0 裁决器 | 汇总V1-V5投票，计算最终置信度 | 纯代码(TypeScript) |
| V1 来源回溯 | fetch source_url，核对事实是否在原文中 | Fetch + AI |
| V2 多来源交叉 | 同一事件**不同独立信息源**是否一致，验证source_url独立性，少数服从多数 | 代码(metric+独立性) + AI(event+转载检测) |
| V3 数值合理性 | 数字量级/范围是否合理，与历史数据对比 | 纯代码 |
| V4 链上锚定 | 声称的链上数据与DeFiLlama/CoinGecko实际数据匹配吗 | 纯代码 |
| V5 时序一致性 | 时间线有无逻辑矛盾 | 代码 + AI(冲突时) |
| V6 时间线归属 | 独立确认事实的时间线归属(B3之后的第二双眼睛) | AI |

**置信度体系**:
- 🟢 **高置信** — 多个**独立信息源**验证一致 + 来源原文匹配 + 数值合理 + 链上锚定（注：转载/引用同一原始报道不算独立）
- 🔵 **中置信** — 单来源但来源已验证 + 数值合理（或有多来源但信息源不独立）
- 🟡 **低置信** — 来源不可用或仅部分验证
- ❌ **拒绝** — 来源原文无依据 / 数值明显错误 / 链上数据不符 / 多源中处于少数方

**硬性否决规则**(任一触发即拒绝):
1. V1来源回溯：原文中找不到依据
2. V3数值合理性：量级明显错误(如$52B写成$520B)
3. V4链上锚定：与链上实际数据严重不符(偏差>20%)
4. V2多源交叉：多来源中处于少数方

被拒绝的事实不进入系统，不显示给用户。每条事实的验证详情可在前端展开查看。

### 自由重组：你可以怎么查

原子事实拆好之后，你可以用**任意维度组合查询**：

```
"给我看所有关于'收入结构'的事实"
→ 从Circle S-1、Tether季报、PayPal财报中抽出所有收入相关事实
→ 跨实体、跨时间、自动并排

"给我看本周所有'具体数字'"
→ 47条信息中的83个metric类型事实全部列出
→ 按实体分组，一目了然

"给我看所有'跨境支付'标签的事实，过去3个月"
→ 时间线视图展示所有跨境支付相关原子事实

"给我看Stripe和Circle的所有'合作关系'类事实"
→ 两个实体间关系的完整事实基础

"给我看本周所有来源是'官方'的事实"
→ 只看一手信源，排除二手解读
```

## 七大杀手功能

### 1. 原子事实 + 自由重组 (核心)

上面已详述。这是产品的底层架构，所有其他功能建立在此之上。

### 2. 事实矛盾检测

系统自动检测不同来源对同一事物的矛盾描述，**标红但不判断谁对谁错**。

```
┌─────────────────────────────────────────────┐
│ ⚠️ 事实矛盾 #17                             │
├─────────────────────────────────────────────┤
│                                             │
│ 关于: Circle 2025年收入                      │
│                                             │
│ 来源A: SEC S-1 Filing (官方)                 │
│ "年收入$1.7B"                               │
│                                             │
│ 来源B: @CryptoAnalyst 推文 (社交)            │
│ "收入约$1.5B"                               │
│                                             │
│ 差异: $200M (11.8%)                         │
│ 可能原因: 时间范围不同? 会计口径不同?         │
│                                             │
│ [查看来源A原文] [查看来源B原文]              │
└─────────────────────────────────────────────┘
```

**实现方式**: 纯规则+AI辅助。
- 对同一实体的同一metric_name，检查不同source的metric_value是否一致
- 数值型：差异超过阈值(如5%)则标记
- 文本型：AI判断两条事实是否在描述同一件事但内容矛盾
- 系统只标记矛盾，不判断谁对

### 3. 知识盲区探测

**核心逻辑：跨实体结构对比。** 不是AI"觉得"哪里有盲区，而是**对比同类实体的事实覆盖结构，缺失的维度就是盲区**。

```
┌─────────────────────────────────────────────────────┐
│ 🔍 知识盲区报告 — 稳定币发行商                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│ 对比: Circle vs Tether vs PayPal(PYUSD) vs Ethena   │
│                                                     │
│             Circle  Tether  PYUSD   Ethena          │
│ 市值         ✅      ✅      ✅      ✅              │
│ 交易量       ✅      ✅      ✅      ✅              │
│ 链分布       ✅      ⚠️(少)  ✅      ✅              │
│ 收入数据     ✅      ✅      ❌      ❌              │
│ 储备构成     ✅      ⚠️(旧)  ✅      ✅              │
│ 用户数       ❌      ❌      ⚠️(少)  ❌              │
│ 合规牌照     ✅      ⚠️(少)  ✅      ❌              │
│ 团队信息     ✅      ❌      ✅      ✅              │
│ 融资历史     ✅      ❌      N/A     ✅              │
│                                                     │
│ 总结:                                               │
│ • "用户数" 维度: 4个实体中3个缺失                    │
│ • Tether: 缺"链分布"详细数据和"团队信息"             │
│ • Ethena: 缺"收入数据"和"合规牌照"信息              │
│                                                     │
│ ⚠️ = 有数据但较少或较旧  ❌ = 无数据                  │
└─────────────────────────────────────────────────────┘
```

**实现方式**: 纯结构化对比，零AI主观。
- 定义每类实体应有的"事实维度模板"（如发行商应有：市值、交易量、链分布、收入、储备...）
- 模板来源于该类别中事实覆盖最全的实体（如果Circle有10个维度的数据，那其他发行商也应该有这10个维度）
- 缺失 = 该实体在该维度的atomic_facts数量为0
- ⚠️ = 有但数量显著低于同类平均，或最新数据超过N天

### 实体档案

每个实体的"活档案"，所有原子事实自动归档，含团队标注。

### 事件时间线

跨周追踪持续事件（Circle IPO、GENIUS Act立法等），每个节点关联具体原子事实。

### 监管追踪器

按国家/地区组织监管事实，维护法案状态流转。

### Twitter Voices

每周采集，按角色分类（VC/KOL/创始人/用户），同时拆成原子事实归入实体档案。

### 周报快照

知识图谱的时间切片，包含：本周Diff、密度异常、矛盾、盲区变化、团队标注汇总。通过邮件+Telegram推送。

### AI对话

基于全部原子事实的RAG对话。System prompt强制只返回事实查询结果，不返回主观判断。
